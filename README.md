# Project Name
React Personal Portfolio Website

## Project Description
This is a responsive and visually appealing personal portfolio website developed using React. The portfolio is designed to showcase my professional profile, technical skills, projects, education, and contact information in a clean and user‑friendly manner. The website follows a light, warm theme and uses reusable React components for better structure and maintainability.
The application is optimized for performance and responsiveness, ensuring a seamless user experience across different devices and browsers.

## Features
- Hero section with name and professional title
- About Me section
- Skills section with styled skill chips
- Projects section with project descriptions
- Education details
- Resume download button
- Contact section with clickable Email, LinkedIn, and GitHub links
- Footer section
- Responsive design (mobile, tablet, desktop)
- Clean and professional UI

## Technologies Used
- React
- JavaScript (ES6)
- HTML5
- CSS3
- Vite
- Git & GitHub

## Installation & Setup
1. Clone the repository from GitHub  
   git clone https://github.com/sakshi-khangar/React-personal-portfolio.git

2. Navigate into the project folder  
   cd My-Portfolio

3. Install all required dependencies  
   npm install

4. Run the project in development mode  
   npm run dev
The application will run on: http://localhost:5173/

## Production Build
To generate an optimized production build of the application, 

run: npm run build

This command creates a production-ready build inside the dist folder.

## Deployment
The project is deployed using Vercel.
Steps followed:
- Pushed the project code to GitHub
- Connected the GitHub repository to Vercel
- Selected Vite as the framework
- Used the following configuration:
- Build Command: npm run build
- Output Directory: dist

## Challenges Faced & Solutions
Challenges:
- Choosing a light but non‑white color theme
- Designing a clean and professional UI layout
- Organizing content using reusable React components
  
Solutions :
- Used warm neutral colors for better visual comfort
- Implemented card‑based layout for clarity
- Separated UI into reusable components
